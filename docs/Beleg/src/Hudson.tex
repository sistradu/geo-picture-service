\section{Hudson}
\label{JumpHudson}
Ein weiterer Bestandteil der Entwicklungsumgebung war der Continuous Integration Server Hudson. Dieser wird über Tomcat gestartet und in Verbindung mit dem Versionsverwaltungssystem bietet es die Möglichkeit automatisiert die Anwendung nach einem Commit neu zu deployen und zu testen. Daraus ergeben sich Vorteile, wie z.B. einer stetig aktuellen und verfügbaren Version der Anwendung, die automatisierte Ausführung von Test über ein Ant-Skript, sowie sofortiges Feedback darüber, ob Integrations-Probleme bestehen. Das Ergebnis eines neuen Buildvorganges von Hudson lässt sich per E-Mail an die Projektmitglieder versenden oder über einen RSS-Feed abrufen.\newline\newline
Die Entscheidung zu Gunsten von Hudson fiel, da in einem Vortrag über die Entwicklung eines Web-Projektes mit diesem System gute Erfahrungen gemacht wurden. Ähnliches gilt auch für dieses, die Installation und Konfiguration erfordert etwas Zeit, wenn man dies zum ersten Mal macht. Allerdings überwiegen die genannten Vorteile. Somit kann die Verwendung von Hudson für ein ähnliches Projekt wie in dieser Belegarbeit empfohlen werden.
