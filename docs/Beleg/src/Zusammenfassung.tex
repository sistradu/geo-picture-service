\chapter{Zusammenfassung}
\label{JumpZusammenfassung}
Basierend auf der Aufgabenstellung wurde das existierende Content Management System Drupal untersucht. Das Ergebnis war, dass nicht alle gestellten Anforderungen damit umgesetzt werden konnten und somit die Erstellung eines neuen Systemes von Nöten war. Die formulierten Wünsche an das System wurden in Pflicht- und Zusatzaufgaben eingeteilt. Nach der Erstellung des Systementwurfes, welcher die geforderten XML-Technologien und Frameworks berücksichtigte, wurde eine Evaluierung von neuen Technologien durchgeführt. Implementiert wurde das Projekt mit Hilfe der IDE IntelliJ. In diesem Zusammenhang wurde eine Evaluierung der Entwicklungsumgebung vollzogen. Als Ergebnis steht ein Software-Projekt, welches die Pflichtanforderungen erfüllt. Gleichzeitig wurden die verwendeten Technologien dahingehend untersucht, ob sie sich zu der Entwicklung einer solchen Anwendung eignen und welche Stärken und Schwächen sie besitzen.\newline\newline
Die Verwendung einer nativen XML-Datenbank in Verbindung mit dem Austausch von Daten über XML-Dateien, eignet sich hervorragend um Webprojekte zu erstellen. Unterstützt durch die Entwicklungsumgebung, die dem Entwickler den Freiraum für Kreativität lässt, da er die Umgebung individuell an seine Bedürfnisse anpassen kann. Diese Freiheit hat einen Preis, da Lizenzkosten anfallen. Die Integration von Primefaces-Komponenten bietet eine einfache Möglichkeit komplexe Anforderungen, wie eine Integration von personalisierten GoogleMaps-Karten, umzusetzen. Allerdings ist die Dokumentation an einigen Stellen lückenhaft und das Verhalten nicht immer so wie erwartet. Trotz dieser Mängel eignet sich Primefaces für die Entwicklung von Webanwendungen, da frühere Projekte gezeigt haben, dass andere Frameworks dieser Art, wie RichFaces oder ICEFaces, ähnliche Schwächen gezeigt haben.
